Introducción al Servicio DHCP

Aprende cómo funciona DHCP, la asignación automática de direcciones IP y la administración centralizada de redes.

Introducción

DHCP (Dynamic Host Configuration Protocol) es un protocolo estándar TCP/IP que automatiza la configuración IP de los dispositivos conectados a una red.

Gracias a DHCP, los equipos reciben automáticamente:

  • Dirección IP
  • Máscara de subred
  • Puerta de enlace
  • Servidor DNS
Esto permite administrar toda la red desde un único servidor, evitando configuraciones manuales en cada ordenador.

Servidor DHCP

El servidor DHCP escucha las solicitudes de los clientes y responde proporcionando automáticamente los parámetros de configuración de red.

Los clientes deben estar configurados con la opción:

Obtener dirección IP automáticamente

Parámetros proporcionados

  • Dirección IP
  • Máscara de subred
  • Puerta de enlace
  • Servidores DNS
  • Otros parámetros adicionales

Ventajas de usar DHCP

  • Evita conflictos de direcciones IP repetidas.
  • Facilita la integración automática de nuevos equipos en la red.
  • Mantiene una administración centralizada.
  • Reduce errores humanos en la configuración.
  • Permite cambios rápidos desde el servidor.

Funcionamiento del Servicio DHCP

El servidor DHCP asigna direcciones IP dentro de un rango previamente configurado.

Si una IP ya está siendo utilizada, el cliente solicitará otra dirección disponible.

Cuando el cliente alcanza el 50% del tiempo de concesión, intentará renovar automáticamente su dirección IP.

Tipos de Mensajes DHCP

DHCPDISCOVER

El cliente busca servidores DHCP disponibles.

DHCPOFFER

El servidor ofrece una configuración IP.

DHCPREQUEST

El cliente acepta la oferta del servidor.

DHCPACK

Confirmación final de la asignación IP.

DHCPDECLINE

El cliente informa que la IP está en uso.

DHCPNACK

El servidor indica que la IP no es válida.

DHCPRELEASE

El cliente libera la dirección IP asignada.

Puerto servidor: UDP 67

Puerto cliente: UDP 68

Conceptos DHCP

Ámbito DHCP

Agrupación administrativa de clientes de una subred que utilizan el servicio DHCP.

Rango DHCP

Conjunto de direcciones IP que el servidor puede asignar.

Ejemplo: 192.168.0.1 - 192.168.0.254

Concesión o Alquiler

Tiempo durante el cual un cliente puede utilizar una dirección IP.

Reserva de Direcciones

Permite asignar siempre la misma IP a un equipo utilizando la dirección MAC.

Exclusiones

Direcciones IP del rango que nunca serán asignadas.

Administración de Direcciones con DHCP

Asignación Manual

El administrador configura manualmente la IP del cliente desde el servidor DHCP.

Asignación Automática

El servidor asigna una IP permanente en el primer contacto.

Asignación Dinámica

La dirección IP se asigna por un tiempo determinado. Una vez expirada, el cliente deberá solicitar una nueva.
