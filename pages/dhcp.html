DHCP: El Protocolo para la Asignación Automática de Direcciones IP

Introducción y Definición

DHCP (Protocolo de Configuración Dinámica de Host) es un protocolo estándar TCP/IP que automatiza la asignación de direcciones IP y otros parámetros de red a los dispositivos al conectarse. Su implementación resuelve el problema de la configuración manual de direcciones IP, evitando errores humanos y simplificando la administración de redes.

Un servidor DHCP proporciona, como mínimo, la dirección IP y la máscara de subred. Opcionalmente, también puede asignar la puerta de enlace (*gateway*) y los servidores DNS.

Funcionamiento Detallado: El Ciclo DORA

DHCP opera bajo un modelo cliente/servidor y sigue un ciclo de cuatro pasos conocido como DORA:

  • Discover (DHCPDISCOVER): El cliente localiza un servidor enviando un mensaje de difusión (*broadcast*).
  • Offer (DHCPOFFER): El servidor responde con una oferta de una dirección IP disponible y otros parámetros de red.
  • Request (DHCPREQUEST): El cliente acepta la oferta enviada por el servidor.
  • Acknowledge (DHCPACK): El servidor confirma la asignación y reserva la dirección IP para el cliente.

Los puertos utilizados son: UDP 67 en el servidor y UDP 68 en el cliente. Es importante notar que si un cliente no logra encontrar un servidor DHCP, no podrá conectarse a la red.

Gestión de Concesiones (Alquiler)

La concesión o alquiler de direcciones es el período durante el cual un cliente puede utilizar la dirección IP asignada por el servidor DHCP.

El proceso de renovación ocurre cuando el cliente alcanza el 50% del período de la concesión. Si la concesión sigue vigente, el cliente puede mantener la misma IP; de lo contrario, debe solicitar una nueva dirección IP.

Ventajas Clave de Implementar DHCP

  • Configuración rápida y automática (*Plug and play*).
  • Evita conflictos de direcciones repetidas (conflictos de IP).
  • Simplifica la administración de la red mediante administración centralizada.
  • Permite realizar cambios de configuración IP desde el servidor sin intervención física en cada equipo.

Administración Avanzada y Métodos de Asignación

Métodos de Asignación

  1. Manual: El administrador configura la IP manualmente en el servidor, usando DHCP solo para entregar el valor.
  2. Automática: El servidor asigna una IP permanente en el primer contacto.
  3. Dinámica: Asigna la IP por un tiempo determinado (concesión); la dirección se revoca al expirar y el cliente debe solicitarla de nuevo.

Conceptos Clave de Administración

  • Ámbito (*Scope*): Agrupamiento administrativo de clientes de una subred.
  • Rango (*Range*): Conjunto de direcciones IP que el servidor puede conceder.
  • Exclusiones: IP que pertenecen al rango, pero que no serán asignadas.
  • Reserva de direcciones IP: Mecanismo para asignar siempre la misma IP a un equipo específico (asociando MAC a IP).
