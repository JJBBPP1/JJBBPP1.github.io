Introducción al Servicio DNS

Aprende cómo funciona el Sistema de Nombres de Dominio, la resolución de nombres y la arquitectura DNS en Internet.

¿Qué es el Servicio DNS?

DNS (Domain Name System) es el sistema encargado de traducir nombres de dominio en direcciones IP dentro de una red TCP/IP.

Gracias al DNS, las personas pueden acceder a sitios web utilizando nombres fáciles de recordar en lugar de direcciones IP numéricas.

Ejemplo:

www.google.com → 142.250.184.196

Espacio de Nombres de Dominio

El DNS utiliza una base de datos distribuida organizada como un árbol jerárquico invertido.

Cada dominio representa una rama dentro de esa estructura.

Raíz

Nivel superior del árbol DNS.

TLD

Dominios de nivel superior como: .com, .org, .es o .net

Subdominios

Especializaciones dentro de un dominio.

blog.empresa.com

Uso del DNS

  • Resolución de nombres: traducir nombres de dominio a direcciones IP.
  • Resolución inversa: obtener el nombre asociado a una IP.
  • Resolución de servidores de correo mediante registros MX.
  • Validación y seguridad mediante SPF y claves de cifrado.

www

Dominio de nivel superior.

Arquitectura DNS

El sistema DNS funciona principalmente utilizando el protocolo UDP y el puerto 53.

Puerto DNS: UDP 53

El sistema se organiza mediante servidores distribuidos que tienen autoridad sobre diferentes dominios y subdominios.

Root Servers

Los servidores raíz son los encargados de dirigir las consultas hacia los dominios correctos.

Proceso de Resolución de Nombres

  1. El cliente solicita resolver un dominio.
  2. El servidor DNS consulta a un root server.
  3. El root server indica qué servidor tiene autoridad.
  4. La consulta continúa hasta encontrar el dominio correcto.
  5. El servidor devuelve la dirección IP al cliente.
blog.empresa.com → 192.168.1.20

Mecanismos de Caché

Los servidores DNS almacenan temporalmente las respuestas para acelerar futuras consultas.

Cada respuesta tiene un tiempo de vida llamado:

TTL - Time To Live

Debido a este sistema de caché, los cambios DNS pueden tardar minutos o incluso días en propagarse completamente.

Correo Electrónico y DNS

El DNS también participa en el envío de correos electrónicos utilizando registros MX.

MX

Define el servidor encargado de recibir correos.

SMTP

Protocolo utilizado para enviar mensajes.

Puerto 25

Puerto utilizado por SMTP.

Tipos de Registros DNS

A

Asocia nombres con direcciones IP.

CNAME

Crea alias para hosts.

NS

Define servidores de nombres.

MX

Servidor de correo electrónico.

PTR

Resolución inversa de IPs.

TXT

Información adicional del dominio.

BIND - Servidor de Nombres

BIND (Berkeley Internet Name Domain) es el software más utilizado para implementar servidores DNS.

Se utiliza ampliamente en sistemas Unix y Linux.

DNS en Redes Locales

  • Mejora la velocidad de acceso mediante caché local.
  • Reduce tráfico hacia Internet.
  • Simplifica la administración de equipos.
  • Permite usar nombres fáciles de recordar.
impresora.oficina.local

Problemas del DNS

  • Al utilizar UDP, algunos paquetes pueden perderse.
  • La propagación de cambios puede ser lenta.
  • Algunos servidores DNS de ISP están mal configurados.
Una solución común es utilizar un servidor DNS local conectado directamente con los root servers.

Herramientas DNS

Existen herramientas para analizar y solucionar problemas DNS.

dig

Herramienta Unix para consultas DNS.

nslookup

Herramienta disponible en Windows y Linux.
